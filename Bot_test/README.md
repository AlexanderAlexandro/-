# Telegram Utility Bot

Личный Telegram-бот на Python для быстрых рабочих задач без базы данных и без внешних AI API.

Бот подходит как компактный личный хаб в Telegram: открыть шаблон, быстро собрать рабочий текст, посчитать метрику или посмотреть короткую памятку.

## Что умеет бот

- показывает готовые рабочие шаблоны;
- собирает большие структурированные тексты по шагам через FSM в памяти;
- после пошаговой сборки отдает не только структуру, но и готовый связный текст, который можно сразу отправить или вставить в документ;
- считает базовые и продуктовые метрики;
- хранит подробные полезные памятки;
- отправляет владельцу отчет после каждого завершенного сценария;
- умеет работать в групповых чатах: собирать текущую переписку за день и по команде владельца отправлять ее в личку;
- показывает владельцу оценку здоровья чата, ключевые сигналы, топ-участников и рекомендации по рабочему ритму;
- позволяет владельцу отправлять сообщение от имени бота в выбранный известный чат;
- сам напоминает в чате, если там не было новых человеческих сообщений больше 3 суток;
- позволяет владельцу включать и выключать автонапоминания отдельно для каждого чата;
- поддерживает короткий набор команд без перегруза: `/start`, `/help`, `/stop`, а для владельца еще `/admin` и `/shutdown`;
- дает отдельный быстрый набор для чатов про лендинги, сайты и запуск стартапа через `/kit`;
- работает локально через `aiogram 3` и `.env`.

## Сценарии использования

### 1. Быстро собрать рабочий текст

Откройте раздел `Шаблоны`, выберите категорию и нужный шаблон. Дальше бот:

- покажет пустую заготовку, если нужно просто скопировать структуру;
- или задаст вопросы по шагам и соберет готовый текст.

Подходит для:

- гипотез;
- драфтов фич;
- брифов лендинга;
- формулировок оффера;
- портрета ICP;
- целей фич;
- аналитических выводов;
- деловых сообщений;
- саммари встреч;
- задач разработчику и дизайнеру;
- апдейтов по запуску стартапа.

На выходе бот показывает:

- готовый текст в естественной формулировке;
- структурированную версию по полям, если нужно быстро отредактировать отдельный блок.

### 2. Быстро сделать расчет

Откройте раздел `Калькуляторы` и выберите нужный сценарий:

- процент от числа;
- рост между двумя числами;
- скидка;
- конверсия;
- ROI;
- CTR;
- CPL;
- retention;
- churn;
- stickiness DAU/MAU;
- ARPU;
- ARPPU;
- LTV;
- CAC;
- AOV;
- payback CAC;
- runway.

Бот последовательно спросит входные значения и сразу покажет аккуратный итог.

### 3. Держать под рукой рабочие памятки

В разделе `Полезное` лежат короткие статические инструкции:

- как поставить задачу разработчику;
- как описать гипотезу;
- как быстро проверить драфт;
- как подготовить саммари встречи;
- как собрать сильный лендинг;
- как сформулировать оффер;
- как описать ICP;
- как писать апдейт по запуску;
- как проверить лендинг перед запуском;
- как провести CustDev-интервью;
- как быстро проверить новый канал;
- как оценить рабочий чат команды.

Каждый блок теперь содержит не только краткий список, но и дополнительные пояснения и мини-чек-листы.

### 4. Быстрый набор для чата про лендинги и стартап

Если бот используется в активном рабочем чате, можно открыть специальный набор через `/kit`.

Там собраны:

- быстрые шаблоны для лендинга, оффера, ICP и апдейта по запуску;
- профильные гайды;
- быстрые переходы к ключевым метрикам: `CAC`, `LTV`, `ROI`, `CTR`, `CPL`, `Runway`.

### 5. Работать с чатами как владелец бота

Если бот добавлен в группу или супергруппу, он может:

- сохранять сообщения, которые реально увидел в чате в течение дня;
- по команде владельца в личке отправлять историю за сегодня;
- показывать владельцу быструю оценку чата за последние 3 суток: активность, решения, блокеры, вопросы, ссылки и топ-участников;
- выделять последние сигналы из переписки: вопросы, договоренности, блокеры и следующие шаги;
- по команде владельца копировать любое отправленное ему сообщение в выбранный чат.
- раз в час проверять активность и писать в чат, если в нем тишина больше 3 суток.

Важно:

- бот не умеет забирать старую историю задним числом через Bot API;
- он может работать только с теми сообщениями, которые получил после добавления в чат;
- для полного логирования бот должен быть администратором в группе или у него должен быть отключен privacy mode через BotFather.
- напоминание о тишине отправляется один раз на период неактивности и сбрасывается только после нового человеческого сообщения.

## Требования

- Python 3.11
- Telegram Bot Token от [@BotFather](https://t.me/BotFather)

## Быстрый запуск

1. Перейдите в папку проекта:

```bash
cd /Users/a.vostretsov/Downloads/Bot_test
```

2. Создайте виртуальное окружение:

```bash
python3.11 -m venv .venv
```

3. Активируйте его:

```bash
source .venv/bin/activate
```

4. Установите зависимости:

```bash
pip install -r requirements.txt
```

5. Создайте `.env` на основе примера:

```bash
cp .env.example .env
```

6. Укажите токен в `.env`:

```env
TELEGRAM_BOT_TOKEN=ваш_токен_из_BotFather
OWNER_TELEGRAM_ID=ваш_telegram_id
INACTIVITY_REMINDER_DAYS=3
INACTIVITY_REMINDER_TEXT=Коллеги, привет.\n\nВ чате давно не было новых сообщений. Хотел уточнить, все ли в порядке и есть ли открытые вопросы, которые стоит обсудить?\n\nЕсли нужно, можно коротко написать статус или зафиксировать, что осталось в работе.
```

`OWNER_TELEGRAM_ID` нужен, если вы хотите получать отчеты после завершения каждого сценария. Если отчеты не нужны, переменную можно оставить пустой.

`INACTIVITY_REMINDER_DAYS` задает срок тишины в чатах, после которого бот напишет напоминание.

`INACTIVITY_REMINDER_TEXT` задает сам текст напоминания. Для переноса строк используйте `\n`.

7. Запустите бота:

```bash
python main.py
```

Если все настроено правильно, бот начнет polling и станет доступен в Telegram.

## Запуск в фоне на Mac

Если хотите, чтобы бот работал даже после закрытия терминала, используйте `launchd`.

В проекте уже подготовлены файлы:

- [scripts/run_bot.sh](/Users/a.vostretsov/Downloads/Bot_test/scripts/run_bot.sh)
- [scripts/install_background_service.sh](/Users/a.vostretsov/Downloads/Bot_test/scripts/install_background_service.sh)
- [launchd/com.bot_test.telegram_utility_bot.plist](/Users/a.vostretsov/Downloads/Bot_test/launchd/com.bot_test.telegram_utility_bot.plist)

### Включить фоновый запуск

Самый быстрый вариант:

```bash
cd /Users/a.vostretsov/Downloads/Bot_test && ./scripts/install_background_service.sh
```

Если хотите пройти шаги вручную:

1. Сделайте скрипт исполняемым:

```bash
chmod +x /Users/a.vostretsov/Downloads/Bot_test/scripts/run_bot.sh
```

2. Скопируйте plist в `LaunchAgents`:

```bash
mkdir -p ~/Library/LaunchAgents
cp /Users/a.vostretsov/Downloads/Bot_test/launchd/com.bot_test.telegram_utility_bot.plist ~/Library/LaunchAgents/
```

3. Подключите агент:

```bash
launchctl bootout gui/$(id -u) ~/Library/LaunchAgents/com.bot_test.telegram_utility_bot.plist 2>/dev/null || true
launchctl bootstrap gui/$(id -u) ~/Library/LaunchAgents/com.bot_test.telegram_utility_bot.plist
launchctl kickstart -k gui/$(id -u)/com.bot_test.telegram_utility_bot
```

После этого бот будет работать в фоне и подниматься после входа в систему.

### Проверить статус

### Посмотреть логи



### Полностью выключить автозапуск


## Как устроен проект

- `main.py` создает `Bot`, `Dispatcher`, in-memory storage и подключает роутеры.
- `config.py` загружает `TELEGRAM_BOT_TOKEN` и `OWNER_TELEGRAM_ID` из `.env`.
- `data/` хранит статические данные отдельно от логики: шаблоны, сценарии вопросов, калькуляторы, тексты интерфейса.
- `services/` содержит бизнес-логику: рендер экранов, сборку итоговых текстов, расчеты, валидацию, отчеты владельцу и хранение истории чатов.
- `handlers/` обрабатывает команды, callback-кнопки, owner-only сценарии и шаги FSM.
- `keyboards/` собирает inline-клавиатуры для меню, разделов и сценариев.
- `utils/` хранит небольшие общие helpers для форматирования, навигации, состояний, безопасного редактирования сообщений и middleware логирования.
- `runtime/` создается автоматически во время работы и содержит служебные файлы логирования чатов.

## Навигация и состояние

- Бот использует `FSM` только в памяти процесса.
- Данные не записываются в базу и не сохраняются между перезапусками.
- Во всех пошаговых сценариях есть кнопки:
  `Назад`, `В главное меню` и переход обратно в раздел.
- После завершения сценария бот показывает кнопки:
  `Собрать заново`, `В шаблоны` или `В калькуляторы`, `В главное меню`.
- После завершения пошагового шаблона или расчета бот может отправить владельцу отчет с пользователем, разделом, названием сценария и итоговым результатом.
- Владелец может открыть `/admin` в личке с ботом и:
  выбрать известный чат, получить историю за сегодня, увидеть оценку чата, посмотреть сигналы из переписки или отправить сообщение от имени бота.

## Как добавить новый шаблон

### Шаг 1. Добавьте описание шаблона

Откройте [data/templates.py](/Users/a.vostretsov/Downloads/Bot_test/data/templates.py) и добавьте новую запись в `TEMPLATES`:

- уникальный `id`;
- `category_id`;
- название `title`;
- описание `use_case`;
- `empty_template`;
- список `sections`, которые определяют структуру итогового вывода.

### Шаг 2. Добавьте пошаговый сценарий

Откройте [data/template_scenarios.py](/Users/a.vostretsov/Downloads/Bot_test/data/template_scenarios.py) и добавьте сценарий в `TEMPLATE_SCENARIOS`.

Для каждого шага укажите:

- `key` поля;
- текст вопроса;
- при необходимости `choices`, если нужен выбор кнопками.

### Шаг 3. При необходимости добавьте “готовый вариант”

Если шаблон должен собираться не только в структуру, но и в естественный итоговый текст, добавьте renderer в [services/template_service.py](/Users/a.vostretsov/Downloads/Bot_test/services/template_service.py):

- создайте приватную функцию вида `_build_my_template_text`;
- подключите ее в словарь `READY_TEXT_BUILDERS`.

Если этого не сделать, бот все равно будет работать и покажет структурированную версию результата через generic renderer.

## Как добавить новый калькулятор

### Шаг 1. Опишите калькулятор

Откройте [data/calculators.py](/Users/a.vostretsov/Downloads/Bot_test/data/calculators.py) и добавьте запись в `CALCULATORS`:

- `id`;
- `title`;
- `description`;
- `formula_hint`;
- список `questions`.

### Шаг 2. Добавьте валидацию

Если калькулятору нужна особая проверка значений, добавьте приватную функцию в [services/calculator_service.py](/Users/a.vostretsov/Downloads/Bot_test/services/calculator_service.py) и зарегистрируйте ее в словаре `VALIDATORS`.

### Шаг 3. Добавьте рендер результата

Создайте приватную функцию рендера, например `_render_my_calc_result`, и добавьте ее в `RESULT_RENDERERS`.

Именно этот слой отвечает за:

- формулу расчета;
- итоговый текст;
- форматирование результата.

## Работа с чатами

### Что умеет бот

- регистрировать группы и супергруппы, в которые его добавили;
- сохранять сообщения, которые он реально получил в этих чатах;
- показывать владельцу историю переписки за сегодня;
- считать индекс активности чата и показывать управленческую оценку по последним сообщениям;
- показывать сигналы из переписки: вопросы, решения, блокеры и следующие шаги;
- копировать любое сообщение из лички владельца в выбранный чат.
- автоматически писать в чат после долгой тишины;
- включать и выключать автонапоминания отдельно для каждого чата.

### Как этим пользоваться

1. Добавьте бота в нужный чат.
2. Сделайте его администратором или отключите privacy mode через BotFather.
3. В личке с ботом отправьте `/admin`.
4. Выберите:
   `История за сегодня`, `Оценка чата`, `Сигналы чата`, `Отправить сообщение в чат` или `Автонапоминания`.

### Команды

Для обычного пользователя нужны только:

- `/start` — открыть главное меню;
- `/kit` — открыть быстрый набор для рабочего чата;
- `/help` — посмотреть короткую справку;
- `/stop` — остановить текущий сценарий и сбросить шаги.

Для владельца дополнительно доступны:

- `/admin` — панель владельца;
- `/shutdown` — полная остановка процесса бота.

Команду `/menu` я оставил как скрытый совместимый алиас, но в пользовательское меню ее не вывожу, потому что для этого уже достаточно `/start` и кнопок навигации.

### Как остановить сценарий или сам бот из Telegram

Если вы хотите просто прервать текущий сценарий и вернуться в спокойное состояние:

1. Откройте личный чат с ботом.
2. Отправьте команду `/stop`.
3. Бот очистит текущий сценарий и покажет главное меню.

Если вы владелец бота и хотите остановить сам процесс:

1. Откройте личный чат с ботом.
2. Отправьте команду `/shutdown`.
3. Бот завершит текущий процесс.

Важно:

- `/stop` доступна всем пользователям и не выключает бота;
- `/shutdown` работает только для владельца;
- после `/shutdown` бот перестанет отвечать, пока вы снова не запустите его через `python main.py`.

### Ограничения

- Через обычный Bot API бот не может забрать старую историю до того, как начал получать сообщения.
- Если история за день не помещается в одно сообщение Telegram, бот отправит ее TXT-файлом.
- Отправка в чат работает только туда, где бот уже состоит и имеет право писать.
- Напоминание о тишине отправляется один раз на период неактивности и сбрасывается после нового человеческого сообщения.
